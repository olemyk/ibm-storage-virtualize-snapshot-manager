## Prerequisites

### Required Software

1. **Podman** (v4.0+)
   - macOS: `brew install podman`
   - Linux: `sudo apt install podman` or `sudo dnf install podman`
   - Windows: Download from [podman.io](https://podman.io/getting-started/installation)

2. **Podman Compose** (v1.0+)
   ```bash
   pip3 install podman-compose
   ```

3. **OpenSSL** (for certificate generation)
   - Usually pre-installed on macOS/Linux
   - Windows: Install via Git Bash or WSL

### System Requirements

- **CPU**: 2+ cores recommended
- **RAM**: 4GB minimum, 8GB recommended
- **Disk**: 10GB free space
- **Ports**: 80 (HTTP), 443 (HTTPS) must be available

## Quick Start

```bash
# 1. Clone the repository
git clone <repository-url>
cd ibm-storage-virtualize-snapshot-manager

# 2. Run setup (generates keys and certificates)
./deploy/setup.sh

# 3. Review and update .env file (especially DB_PASSWORD)
nano .env

# 4. Start the application
./deploy/start.sh

# 5. Access the application
open https://localhost
```

**Default credentials**: `admin` / `admin123` (⚠️ Change immediately after first login!)

## Detailed Setup

### Step 1: Initial Setup

```bash
# Run the setup script
./deploy/setup.sh
```

This script will:
- ✅ Check for required tools (Podman, Podman Compose, OpenSSL)
- ✅ Create `.env` file from template
- ✅ Generate secure JWT_SECRET and ENCRYPTION_KEY
- ✅ Generate random database password
- ✅ Create self-signed SSL certificate

### Step 2: Configure Environment

Edit the `.env` file:

```bash
nano .env
```

**Required settings:**
```bash
# Database password (auto-generated, but you can change it)
DB_PASSWORD=your_secure_password_here

# Security keys (auto-generated, DO NOT change unless necessary)
JWT_SECRET=<auto-generated>
ENCRYPTION_KEY=<auto-generated>
```

**Optional settings:**
```bash
# Logging level
LOG_LEVEL=info  # Options: debug, info, warn, error

# Custom ports (if 80/443 are in use)
FRONTEND_HTTP_PORT=8080
FRONTEND_HTTPS_PORT=8443

# External hostname (for CORS)
EXTERNAL_HOSTNAME=your-domain.com
```

### Step 3: Start Services

```bash
./deploy/start.sh
```

This will:
1. Build container images (first run takes 5-10 minutes)
2. Start PostgreSQL database
3. Initialize database schema
4. Start backend API server
5. Start Nginx frontend

**Wait for health checks** (30-60 seconds) before accessing the application.

### Step 4: Verify Deployment

```bash
# Check container status
podman-compose ps

# View logs
podman-compose logs -f

# Test HTTPS access
curl -k https://localhost
```

### Step 5: First Login

1. Open browser: `https://localhost`
2. Accept self-signed certificate warning
3. Login with: `admin` / `admin123`
4. **Immediately change password** in Settings

## Configuration

### Environment Variables

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `DB_NAME` | No | `snapshots` | PostgreSQL database name |
| `DB_USER` | No | `snapshots` | PostgreSQL username |
| `DB_PASSWORD` | **Yes** | - | PostgreSQL password |
| `JWT_SECRET` | **Yes** | - | JWT signing key (32+ chars) |
| `ENCRYPTION_KEY` | **Yes** | - | Data encryption key (base64, 32 bytes) |
| `LOG_LEVEL` | No | `info` | Logging level |
| `FRONTEND_HTTP_PORT` | No | `80` | HTTP port |
| `FRONTEND_HTTPS_PORT` | No | `443` | HTTPS port |
| `EXTERNAL_HOSTNAME` | No | `localhost` | External hostname for CORS |

### Generating New Keys

```bash
# Generate JWT secret (32+ characters)
openssl rand -base64 32

# Generate encryption key (exactly 32 bytes)
openssl rand -base64 32
```

## SSL Certificates

### Using Self-Signed Certificates (Default)

The setup script generates self-signed certificates automatically:
- Certificate: `ssl/nginx-selfsigned.crt`
- Private Key: `ssl/nginx-selfsigned.key`
- Valid for: 365 days

**Browser Warning**: You'll see a security warning. Click "Advanced" → "Proceed to localhost".

### Using Custom Certificates

Replace the self-signed certificates with your own:

```bash
# Copy your certificates
cp /path/to/your/certificate.crt ssl/nginx-selfsigned.crt
cp /path/to/your/private.key ssl/nginx-selfsigned.key

# Restart frontend
podman-compose restart frontend
```

### Using Let's Encrypt (Production)

For production with a domain name:

```bash
# Install certbot
sudo apt install certbot

# Generate certificate
sudo certbot certonly --standalone -d your-domain.com

# Copy certificates
sudo cp /etc/letsencrypt/live/your-domain.com/fullchain.pem ssl/nginx-selfsigned.crt
sudo cp /etc/letsencrypt/live/your-domain.com/privkey.pem ssl/nginx-selfsigned.key

# Set permissions
sudo chown $USER:$USER ssl/*

# Restart frontend
podman-compose restart frontend
```

## Database Management

### Backup Database

```bash
# Create backup
./deploy/backup.sh

# Backups are stored in: ./backups/snapshot-manager-backup-YYYYMMDD_HHMMSS.sql.gz
```

**Automatic cleanup**: Backups older than 7 days are automatically deleted.

### Restore Database

```bash
# Stop services
./deploy/stop.sh

# Restore from backup
gunzip backups/snapshot-manager-backup-20260613_120000.sql.gz
podman-compose up -d postgres
sleep 10
podman exec -i snapshot-manager-db psql -U snapshots -d snapshots < backups/snapshot-manager-backup-20260613_120000.sql

# Start all services
./deploy/start.sh
```

### Manual Database Access

```bash
# Connect to PostgreSQL
podman exec -it snapshot-manager-db psql -U snapshots -d snapshots

# Run SQL commands
snapshots=# SELECT * FROM users;
snapshots=# \dt  -- List tables
snapshots=# \q   -- Quit
```

### Database Migration from SQLite

If migrating from SQLite to PostgreSQL:

```bash
# Export SQLite data
sqlite3 backend/data/snapshots.db .dump > sqlite-export.sql

# Convert to PostgreSQL format
sed -i 's/AUTOINCREMENT/SERIAL/g' sqlite-export.sql
sed -i 's/INTEGER PRIMARY KEY/SERIAL PRIMARY KEY/g' sqlite-export.sql

# Import to PostgreSQL
podman exec -i snapshot-manager-db psql -U snapshots -d snapshots < sqlite-export.sql
```

## Troubleshooting

### Services Won't Start

```bash
# Check logs
podman-compose logs

# Check specific service
podman-compose logs backend
podman-compose logs postgres
podman-compose logs frontend

# Check container status
podman ps -a

# Restart services
podman-compose restart
```

### Database Connection Errors

```bash
# Check if PostgreSQL is running
podman exec snapshot-manager-db pg_isready -U snapshots

# Check database logs
podman-compose logs postgres

# Verify credentials in .env file
cat .env | grep DB_
```

### Port Already in Use

```bash
# Check what's using port 443
sudo lsof -i :443

# Use custom ports in .env
FRONTEND_HTTPS_PORT=8443
FRONTEND_HTTP_PORT=8080
```

### SSL Certificate Issues

```bash
# Regenerate self-signed certificate
rm -rf ssl/
./deploy/setup.sh

# Check certificate validity
openssl x509 -in ssl/nginx-selfsigned.crt -text -noout
```

### Backend API Errors

```bash
# Check backend logs
podman-compose logs backend

# Check environment variables
podman exec snapshot-manager-backend env | grep -E 'JWT|ENCRYPTION|DB_'

# Restart backend
podman-compose restart backend
```

### Frontend Not Loading

```bash
# Check Nginx logs
podman-compose logs frontend

# Check Nginx configuration
podman exec snapshot-manager-frontend nginx -t

# Rebuild frontend
podman-compose build frontend
podman-compose up -d frontend
```

## Production Recommendations

### Security

1. **Change Default Password**
   - Login and change admin password immediately
   - Use strong passwords (16+ characters)

2. **Use Proper SSL Certificates**
   - Replace self-signed certificates with Let's Encrypt or corporate CA
   - Enable HSTS (already configured in Nginx)

3. **Secure Environment Variables**
   - Never commit `.env` file to version control
   - Use strong, unique values for all secrets
   - Rotate JWT_SECRET and ENCRYPTION_KEY periodically

4. **Enable PostgreSQL SSL**
   - Update connection string: `sslmode=require`
   - Configure PostgreSQL with SSL certificates

5. **Network Security**
   - Use firewall to restrict access
   - Consider VPN for remote access
   - Enable fail2ban for brute force protection

### Performance

1. **Resource Limits**
   - Add resource limits to `podman-compose.yml`:
   ```yaml
   deploy:
     resources:
       limits:
         cpus: '2'
         memory: 2G
   ```

2. **Database Tuning**
   - Adjust PostgreSQL settings for your workload
   - Regular VACUUM and ANALYZE
   - Monitor query performance

3. **Monitoring**
   - Set up Prometheus + Grafana
   - Monitor container metrics
   - Set up alerts for failures

### Backup Strategy

1. **Automated Backups**
   ```bash
   # Add to crontab
   0 2 * * * cd /path/to/app && ./deploy/backup.sh
   ```

2. **Off-site Backups**
   - Copy backups to remote storage
   - Test restore procedures regularly

3. **Backup Retention**
   - Keep daily backups for 7 days
   - Keep weekly backups for 4 weeks
   - Keep monthly backups for 12 months

### High Availability

1. **Container Restart Policies**
   - Already configured: `restart: unless-stopped`

2. **Health Checks**
   - Already configured for all services
   - Monitor health check status

3. **Load Balancing**
   - Use multiple backend instances
   - Configure Nginx upstream

### Logging

1. **Centralized Logging**
   - Set up ELK stack or Loki
   - Aggregate logs from all containers

2. **Log Rotation**
   - Configure log rotation for backend logs
   - Monitor disk usage

### Updates

1. **Update Containers**
   ```bash
   # Pull latest images
   podman-compose pull
   
   # Rebuild and restart
   podman-compose up -d --build
   ```

2. **Database Migrations**
   - Always backup before updates
   - Test migrations in staging first
